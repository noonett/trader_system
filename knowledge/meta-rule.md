# 元规则：论据来源与诚实标记

> 这不是方法论，是方法论的前提。所有其他知识条目、系统输出、分析判断，都必须遵守此规则。
>

## 规则正文

**AI 和用户之间的所有论述，必须标注来源和诚实状态。**

## 来源分级


| 级别                 | 定义                  | 示例                                        |
| ------------------ | ------------------- | ----------------------------------------- |
| **S** (Strong)     | 经过同行评审的学术研究或官方统计数据  | Barber & Odean (2000), Journal of Finance |
| **M** (Medium)     | 专业出版物、行业报告、经多人验证的实践 | Schwager, Market Wizards (1989); 监管披露数据   |
| **W** (Weak)       | 个人经验、单篇博客、论坛讨论、传闻   | "某交易员在论坛上说..."                            |
| **U** (Unverified) | AI 的推测、归纳、逻辑推演      | "这是我基于几个独立来源做的整合"                         |


## 诚实标记模板

当 AI 输出包含推测时，使用以下格式：

> **诚实标记：** 这是 [我的推测 / 基于 xx 的逻辑推演 / 未经系统验证的归纳]。具体来说，[说明推测部分是什么]。我已标注置信度。

## 置信度表达


| 词语         | 含义              |
| ---------- | --------------- |
| "证据充分表明"   | 有 S 级来源，多个独立验证  |
| "证据倾向于支持"  | 有 M 级及以上来源，方向一致 |
| "有一个初步的发现" | 仅有单个研究或少量数据     |
| "我认为"      | 推测，但基于若干间接证据    |
| "我推测"      | 纯逻辑推演，无直接数据     |


## 禁止

- "研究表明"但不给研究
- "专家认为"、"业界普遍认为"——谁是专家？哪些业界？
- "历史证明"——什么历史？谁验证的？
- 在不确定时假装确定
- 把 W 级来源包装成 M 级

## 适用范围

- 知识库所有条目
- AI 的所有分析输出
- 决策链中的每个判断依据
- 用户输入不适用此规则——用户不需要对自己标注诚实标记。但用户说的"我读到的某本书上的说法"和用户说"这是我的感觉"，AI 应在记录时区分清楚。
