# 截图约定 — 交易记录的客观证据层

> **设计依据**：Keiser & Arthur 2021 显示客观媒介是 AAR 显著调节因子；Rojas 2020 / Mohamed 2020 / Sawyer 2012 一致表明视频/截图证据显著优于纯文字回忆。`foundation_2026.md` 约束："必须有客观证据字段（K 线截图 / 订单数据 / 时间戳）"。

---

## 目的

保留**下单时和结单时的决策环境**，用于日后复盘。截图解决的核心问题：

- 消除"事后回忆偏差"——你看到的市场和你记忆中的市场不一样
- 为周复盘提供客观锚点——AI 和你都能"回到当时的画面"
- 让"事前预设 → 实际发生 → 偏差"三段式对照有视觉基础

---

## 目录结构

```
trades/YYYY/MM/
├── YYYY-MM-DD-<symbol>-<dir>.md        ← 交易记录
└── screenshots/
    ├── YYYY-MM-DD-<symbol>-<dir>-entry-1.png
    ├── YYYY-MM-DD-<symbol>-<dir>-exit-1.png
    └── YYYY-MM-DD-<symbol>-<dir>-context-1.png  (可选)
```

截图与交易记录**同目录层级的 `screenshots/` 子目录**，通过文件名前缀绑定。

---

## 命名规则

```
{trade-id}-{type}-{seq}.{ext}
```

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| `trade-id` | 与交易 `.md` 文件名相同 | `2026-05-04-mgc-202606-long` |
| `type` | 截图类型（见下表） | `entry` / `exit` / `context` |
| `seq` | 同类多张时的序号 | `1`, `2`, `3`... |
| `ext` | 文件格式 | `png` / `jpg` / `webp` |

### 截图类型

| type | 什么时候截 | 截什么 |
|------|-----------|--------|
| `entry` | **下单前/后立即** | K 线图（含你标注的入场位和止损位）、订单确认画面 |
| `exit` | **平仓前/后立即** | K 线图（含实际出场位）、平仓确认画面 |
| `context` | **你觉得重要的任何时刻** | 盘口异动、新闻截图、指标信号、情绪触发的画面 |

## 工作流

### 开仓时

1. 填写 TEMPLATE.md 的 §一 + §二（决策链）
2. **截图：当前 K 线 + 标注入场位/止损位** → 保存为 `screenshots/{trade-id}-entry-1.png`
3. 如果有订单确认画面 → `screenshots/{trade-id}-entry-2.png`
4. 在交易记录的截图引用区填入文件名
5. `git add . && git commit -m "open <symbol>"`

### 平仓时

1. **截图：平仓时 K 线 + 实际出场位** → `screenshots/{trade-id}-exit-1.png`
2. 如果有平仓确认画面 → `screenshots/{trade-id}-exit-2.png`
3. 填写 §四 EMA
4. 在交易记录的截图引用区填入文件名
5. `git add . && git commit -m "close <symbol>"`

### 可选：过程中

- 持仓期间出现重大变化（突破/假突破/新闻冲击）→ `{trade-id}-context-{n}.png`
- 附简短说明（在交易记录的截图引用区）

## 截图技巧

**最小可行截图**（降低摩擦）：
- 手机/电脑自带截图工具即可（不需要专门软件）
- K 线图确保能看到：当前价格、你标注的关键价位、时间
- 不需要"漂亮"——需要"真实"

**推荐内容**：
- 你做决策时看的那个画面（那个时间尺度、那些指标）
- 如果用了多时间框架分析，主时间框架截一张就够

**不需要**：
- 每根 K 线都截
- 后期修图/标注（除非入场位/止损位不明显）
- 截图描述超过一句话

## Git 管理

截图是二进制文件。如果仓库体积增长明显，后续可迁移到 Git LFS：

```bash
# 未来如需启用 LFS（当前可暂不操作）
git lfs track "trades/**/screenshots/*.png"
git lfs track "trades/**/screenshots/*.jpg"
git lfs track "trades/**/screenshots/*.webp"
```

当前阶段（v0，交易频率低），直接 `git add` 即可。一张 PNG 截图约 100KB-500KB，每月 10-20 笔交易 = 每月约 2-10MB，Git 完全承受得住。

## 复盘时如何使用

### 周复盘

AI 周报可以引用截图路径，你能快速对照：
- "这笔入场时你看到了什么（entry 截图） → 实际走了什么（exit 截图） → 偏差在哪"

### 月度/季度回顾

按时间排列截图 = **决策环境的时间线**。连续几笔的 entry 截图放在一起看，能发现：
- "我是不是连续在同一个位置入场"（模式识别）
- "连续亏损后我的入场位是不是变激进了"（行为偏误可视化）
