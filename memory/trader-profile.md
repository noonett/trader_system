## Phase 1 + Phase 2 期间观察到的认知特征（v2 新增；U 级，基于 review 互动模式归纳）

> **诚实标记**：以下是 AI（Claude）在 2026-05-03 至 2026-05-05 多轮 review 互动中观察到的模式。未经用户主动确认，是 U 级归纳。用户读到时可以反驳/修正——profile 不是给用户的判决，是 σ 系统的可调参数（影响哪些 caveat 应该被加权 / 哪些 risk 应该被监控）。

### 1. 偏好诚实优于速度（强信号）

- 5 轮 Phase 1 review 迭代（v1 → v2 → v3 → v4 → v5）每次都接受了"再做一轮 review"的额外成本
- 在 v3 时主动指出"Cursor first 隐含假设"（这是用户自己的元认知 — 看到 AI 的 productization-mode 偏见并校正）
- 拒绝了 AI 提议的"直接出方案不必 review"的捷径多次

### 2. 自评不在极端高风险（用户主动声明 2026-05-05）

- **重要**：这是用户**自评**，临床证据上自评在高风险人群中可靠性有限（DSM-5 problem gambling、Sbarra 2013 等）；AI 在 entry_form_research §6.1 已显式分层为 N=1 自检（不作 lockout gate）+ 状态性风险防御保留
- 在风险监控层面：**不需要**默认接入 PGSI/PHQ-9 + GAD-7 lockout；**需要**在状态性触发条件出现时自动提示（连续亏损 / 睡眠剥夺 / 重大生活事件）

### 3. 完美主义风险中等（U 级，可证伪）

- 倾向"一次性整合所有 review 意见"（Phase 1 v5 14 条 + Phase 2 v2 24 条全采纳）
- 对应风险：DSM-5 problem gambling #2 戒断样反应（"不能完成全部就焦躁"）+ 制度化反思退化为表演性合规（Beighton 2018）
- 监控指标：复盘是否退化为"打勾任务"（每 3 月做一次 meta-audit，详见 v5 §三.8 复盘工具约束）

### 4. 偏好结构化 + 学术化语言（强信号）

- 多次主动要求加 caveat、加诚实标记、加修订记录
- 接受重型 review 流程（multi-model + first-pass + 5 视角并行）
- 启发：σ 系统的 UI 措辞**不需要软化**（"诚实标记"对此用户接受度可能高于一般中文用户；R05-China review 的"像跟法官汇报"反馈对此用户的适用性需 N=1 验证）

### 5. 工具试用倾向（弱信号，2026-05-05 graphify 安装事件）

- 安装了 graphifyy + 两条 cursor rules（karpathy + σ × karpathy precedence）
- 但**graphify 对此仓库价值低**（8 nodes / 10 edges，主要 markdown 不被 parse）
- 启发：可能存在"被新工具吸引但未做需求-能力匹配检查"的工程偏好。监控：未来若再装新工具，是否先做 fit 评估

### 6. 对工作流分层的接受度（强信号）

- 接受 Plan → Design → Action 三阶段
- 接受"一个 PR 一个 logical change"约定（4 个 PR 都遵守了）
- 接受"等外部 review 的成本 vs 直接 merge"的取舍（最终选 b 直接 merge，不强求完美）
