## TL;DR — 一句话结论

**"寻找强 conviction"是新手交易员中极常见的、可识别的行为模式**——核心特征是"先有重仓 / 赚大钱的情绪需求 → 再去找支持这个需求的故事"，而不是健康路径的"事实调研 → 概率判断 → 仓位分配 → 偶尔出现高 conviction 机会"。**σ 系统对此的设计响应是"对照镜"而非"教化"**：把模式从无意识层带到意识层，剩余的判断权留给用户。本笔记记录了 σ 系统在 2026-05-18 完成的首次"对照镜"对话，并把方法论沉淀为可复用框架。

## 一、概念定义

### 1.1 conviction（信念 / 确信）

投资语境下特指"我相信这个判断对到敢用真金白银去赌"的心理状态。

- **high conviction trade**：你最有把握、敢重仓的那笔
- **conviction sizing**：按确信度定仓
- **strong conviction**：强烈的、单方向的、不接受反方证据的"我相信"

**conviction 本身不是坏东西**——巴菲特、彼得·林奇都强调要在自己最确信的事上重仓。问题在因果方向。

### 1.2 健康路径 vs "寻找 conviction" 路径

| 健康路径 | "寻找 conviction" 路径 |
|---|---|
| 1. 事实层调研（多源、多角度）| 1. 情绪需求出现（"我想下重仓 / 想赚大钱"）|
| 2. 得出概率判断（胜率 × 盈亏比）| 2. 寻找一个能支持这个情绪的**故事** |
| 3. 按概率分配仓位（半 Kelly）| 3. 把故事包装成"我已经确信" |
| 4. **偶尔**出现高 conviction 的机会 | 4. 跳过事实层验证，直接下重仓 |
| 因果方向：事实 → 行动 | 因果方向：情绪需求 → 寻找事实（**反向**）|

### 1.3 心理学根源（M-S 级文献）

| 概念 | 来源 | 解释 |
|---|---|---|
| **不确定性厌恶**（uncertainty aversion）| Ellsberg 1961, *QJE* | 人对"我不知道"的状态有生理性不适，倾向选择"已知风险"而非"未知风险" |
| **动机性推理**（motivated reasoning）| Kunda 1990, *Psych Bulletin* | 当我们想要某个结论时，会无意识地用更宽松的标准接受支持它的证据、更严格的标准拒绝反方 |
| **确认偏误**（confirmation bias）| Wason 1960, Nickerson 1998 | 寻找/记住/接受支持已有判断的信息 |
| **新手交易员心理**（normal trader psychology）| Brett Steenbarger 《Daily Trading Coach》2009 | 归类为新手常见模式，非病理 |

**核心机制**：三者叠加 = "先想要一个高 conviction 的故事 → 被自动满足"。

## 二、模式的几何特征（识别清单）

> 这是 AI 在对话中识别模式的可操作清单——不是诊断量表，是模式识别启发式

### 2.1 措辞特征（最直接的信号）

出现以下任一类措辞**即触发对照镜机制**：

- **强度词**：梭哈 / all-in / 重仓 / 满仓 / 翻倍 / 必涨 / 必跌
- **价值断言**：巨大投资价值 / 巨大投资潜力 / 历史性机会 / 千载难逢
- **行动强度**：定投 + 加仓 + 不卖 / 跌了再买
- **反问式确认**："对吧" / "是不是" / "我没看错吧"

### 2.2 行为特征（更深一层的信号）

- **方向不重要**（看多 / 看空都可以满足这个心理需求）
- **强度重要**（必须是"梭哈 / 巨大 / 重仓"，不能是"小仓试探 / 等等看"）
- **范围会扩散**（A 找不到 → B → 整个板块 → 整个赛道）
- **对反方证据的反应**是"换标的 / 换方向"，**不是"调整仓位 / 不参与"**
- 论据来源是**外部转述**（"我收到一个论断" / "群里说" / "大 V 说"）

### 2.3 不构成模式的反例（避免误判）

以下情况**不**应触发对照镜机制：
- 用户在做主题调研但明确说"只是了解 / 没打算实操"
- 用户问"为什么 X 涨" / "X 的基本面如何"（中性求知）
- 用户主动列出**多空两方**论据询问 AI 的看法（已具备元认知）
- 用户问的是**已持有标的的减仓 / 止损**（防守性问题，与寻找 conviction 反向）

## 三、σ 系统的设计响应（"对照镜"角色，非"教化"）

### 3.1 角色定位

按 [foundation §三.2 反馈组件角色](../foundation_2026.md) + [.cursor/rules/sigma-meta-rule-precedence.mdc](../../.cursor/rules/sigma-meta-rule-precedence.mdc)：

> AI 不是教练，也不是分析师。是 **对照镜** + **守门人**。
> - 对照镜：把你的状态反馈给你看
> - 守门人：在规则违反时拉回

**对照镜 ≠ 评判 ≠ 诊断 ≠ 禁止**。是把无意识模式带到意识层，让用户自己做下一步选择。

### 3.2 对照镜的输出格式（4 步）

当 AI 识别到 §二 模式特征时，标准响应路径：

1. **不否定也不配合**：不说"你这样想是错的"也不说"对，X 确实是巨大机会"——两者都跳过了对照镜环节
2. **拆解措辞**：把"巨大潜力 / 梭哈"拆成 3-5 个独立问题（见 [notes/19 §七 "巨大投资潜力 σ 翻译表"](19_alpha_china_ai_manufacturing_2026.md)）
3. **回放证据**：如果是反复出现的模式，把过去 N 次出现的具体语境列给用户看（让证据说话，不让 AI 说话）
4. **留出反驳权**：明确说"我可能看错 / 这不是诊断 / 你可以反驳"——保留用户的元认知主权

### 3.3 落地组件

| 组件 | 位置 | 作用 |
|---|---|---|
| **画像记录** | [traders/{id}/profile/trader-profile.md](../../traders/default/profile/trader-profile.md) "认知特征" 节 | 跨会话持久化（AI 重启后不丢失观察）|
| **风控触发** | [knowledge/risk_rules.md §七 危险信号清单](../../knowledge/risk_rules.md) | 把对话观察固化为下次触发的硬约束 |
| **方法论笔记** | research/notes/20_*.md（本笔记）| 系统设计与未来其他交易员复用 |
| **决策链 5 问** | [foundation §三.13](../foundation_2026.md) | 触发模式后的"30 秒自我对话"工具 |

### 3.4 边界（必须明示）

- **不强行阻断**：用户仍可问"巨大潜力"类问题，AI 仍提供事实层调研——只是输出格式必须包含拆解
- **不假装专业心理评估**：σ 系统不是医疗器械（见 [risk_rules §四](../../knowledge/risk_rules.md)）；模式识别是工程级启发，不是临床诊断
- **用户始终有反驳权**：画像不是单向判决，用户可以修正——但 σ 元规则要求**保留原观察 + 加修订记录**（不允许单向"清除"）

## 四、案例参考：2026-05-18 首次对照镜对话

> 完整时间线与具体证据已在 [traders/default/profile/trader-profile.md §7](../../traders/default/profile/trader-profile.md) 记录，本节只做方法论摘要

### 4.1 触发情景

- 时间窗：2026-05-10 至 2026-05-18（8 天，5 次 α 调研对话）
- 触发词：5/10 "梭哈+定投+巨大投资价值" + 5/18 "巨大投资潜力"
- 行为路径：单股 → 翻转方向 → 题材股 → 整个赛道
- 范围扩散度：从 2 只个股扩散到整个 AI+制造赛道

### 4.2 AI 响应路径

1. **第 5 次对话末尾**（2026-05-18 主题调研之后）：在标准 α 调研输出之外，**单独**列出"现在累积的未处理事项"，**最后一条**写："你近 5 次问题形成'寻找强 conviction'模式。如需，我可以单独写一篇 σ 系统反馈给你的模式识别短笔记"——**不强加**，提供选项
2. **用户追问**（2026-05-18）："寻找强 conviction 是什么"——表明用户接受这是值得讨论的话题
3. **AI 详细解释**：概念定义 + 模式特征 + 用证据回放（不用 AI 主观判断）+ 心理学根源 + σ 系统为何关心 + 回到正轨的方法 + **明示边界与反驳权**
4. **用户接受**（2026-05-18）："你说的有道理，帮我加入模式识别吧"——主动授权落地
5. **AI 实施最小落地**：trader-profile.md 加第 7 项 + risk_rules.md §七 加一条 + 本方法论笔记

### 4.3 这次对话验证了什么

- ✅ **对照镜角色可工作**：用户主动接受反馈而非防御
- ✅ **拆解 + 证据回放 > 直接评判**：让证据说话比 AI 给结论更有说服力
- ✅ **明示反驳权降低用户防御感**：用户在被告知"我可能看错"后反而更接受观察
- ⚠️ **U 级未验证**：5 次对话样本量小；本次用户的"接受度"可能本身就是 §2.4 "对工作流分层接受度高" 的延伸，不代表其他用户也会接受
- ⚠️ **后续监控点**：用户接受 ≠ 模式消失。下一次"巨大"措辞出现时，AI 应再次对照（看用户反应是"已自觉拉回" vs "仍触发"），用以校准模式识别的有效性

## 六、对未来 σ 系统迭代的建议（不是本 PR 范围）

> 仅作待办列表，不在本 PR 实施。需用户单独决定是否拍板

1. **多用户化**：当 σ 系统支持多个 trader 时，每个 trader 的"寻找 conviction"模式可能不同。`trader-profile.md` 需要支持模式严重程度的可配置阈值。
2. **量化追踪**：当前是定性观察。未来可考虑加"措辞频率 / 后续是否真实操"两个量化指标（U 级初步设计；统计显著仍需 ≥ 380 次对话样本）。
3. **AI prompt 工程**：本笔记 §3.2 的"对照镜 4 步输出格式"可以提取为 [sigma/templates/ai-roles/](../../sigma/templates/) 下的可调用模板，让所有 AI 对话都遵循同一格式。
4. **集成到决策链 5 问的预热版**：把 [今天对话](../../traders/default/profile/trader-profile.md) 提到的"30 秒自我对话 4 问"正式加入决策链作为 Q0（前置自检）。
5. **与 clinical_self_check 协同**：寻找 conviction 模式 + 临床自检的特定项（DSM-5 problem gambling #2/#3/#4）有重叠——需要明确"哪个层面分类、哪个先触发"。
