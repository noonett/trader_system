## 总体判断

这份 `foundation_2026.md` 最大优点是：它主动暴露反证据，把“训练成为长期盈利交易者”从隐性承诺改成“有护栏的 aspirational goal”。这比原系统更诚实。

**诚实标记：** 这是我基于 `foundation_2026.md` 全文结构、项目 `meta-rule.md`、以及 5 份 review prompt 的逻辑综合判断。

但作为后续 Phase 2 的“宪法”，它仍有 5 类高优先级风险：

1. **证据外推过强**：不少 S 级组件来自教育、mHealth、军事 AAR、健康行为或监管实验，迁移到主动交易场景仍需标 U/M 边界。
2. **群体混合过宽**：日内交易者、CFD、加密散户、主动基金、长期股票投资者被放进同一“散户失败”叙事，容易过度简化。
3. **Plan / Design 边界仍有残留混用**：字段数、100% ETF、同时挂止损、技术性禁入等有时仍像具体方案。
4. **临床安全层不够硬**：反刍、强迫、焦虑、赌博倾向、金钱压力人群需要更明确的禁忌症与退出提示。
5. **普通用户入口仍不清晰**：如果普通人读完的结论是“那我直接买 ETF 算了”，系统需要解释“为什么还要训练”和“训练只占哪一小部分资金/精力”。

# 视角 1：行为金融学 / 实验经济学学者

## 1.1 “全球散户 71-99% 亏损是 S 级事实”建议降为分层表述

**位置**：`foundation_2026.md:48-58`, `foundation_2026.md:288-289`, `foundation_2026.md:322`  

文档把巴西日内股票/期货、台湾日内交易者、印度 F&O、ESMA CFD、加密散户放在同一张表中，然后在后文形成“全球散户 71-99% 长期亏损”的总叙事。这个方向大概率正确，但“全球散户”过宽。

建议改成：

> 高杠杆 / 短周期 / 衍生品 / 高频交易型零售参与者，在多个市场样本中呈现极高亏损率；但这个证据不能直接外推到所有“普通长期投资者”或所有“主动交易训练者”。

**证据等级**：M/S 混合。  
**诚实标记：** 目标文档引用的 Chague、Barber/Odean、SEBI、ESMA 等方向上支持“高频零售交易者多数亏损”；但“全球散户”作为统一总体是我的统计口径担忧，需要逐篇核验样本定义。

## 1.2 SPIVA 支持“被动基线”，但不直接支持“长期财富资金 100% 被动”

**位置**：`foundation_2026.md:144-146`, `foundation_2026.md:413-416`  

SPIVA 能支持“主动基金长期跑赢指数很难”和“被动投资应作为默认基线”。但“长期财富资金 100% 配置被动 ETF”涉及资产配置、现金流需求、税务、币种、市场可得性、家庭风险承受能力，超出了 SPIVA 本身。

建议改为：

> 长期财富资金以被动低成本指数方案作为默认基线；任何主动偏离必须写明理由、风险预算和退出条件。

**证据等级**：S 支持“被动基线优于多数主动基金”；U 支持“100% ETF 适合具体用户”。  
**诚实标记：** 这是基于 SPIVA 证据适用范围的逻辑推演。

## 1.3 “Implementation intentions d=.27-.66”不能直接推出“交易决策链必然有效”

**位置**：`foundation_2026.md:196`, `foundation_2026.md:215`, `foundation_2026.md:302`, `foundation_2026.md:374`  

文档正确识别 if-then plans 是强证据组件。但从“跨领域元分析有效”到“交易开仓前决策链有效”，中间有迁移风险：交易是高情绪、高金钱反馈、高不确定性、低效度环境。

建议加一句：

> Implementation intentions 是本系统选择 if-then 决策链的证据基础；其在交易下单场景中的实际效果仍属于迁移假设，需用个人级 N-of-1 数据验证。

**证据等级**：S 支持 implementation intentions 一般有效；U 支持其交易场景迁移。  
**诚实标记：** 这是基于跨领域证据迁移边界的逻辑推演。

## 1.4 “55% 胜率需约 380 笔达 95% 置信”应写明假设

**位置**：`foundation_2026.md:166-168`, `foundation_2026.md:394-399`, `foundation_2026.md:437`  

样本量诚实非常重要。但“55% 胜率需要 ~380 笔达 95% 置信”依赖检验对象：是检验胜率 > 50%，还是估计置信区间半宽？是二项分布独立同分布假设，还是考虑交易相关性？

建议补充：

> 该样本量估计基于二项分布、独立交易、以 50% 为基准的近似检验；真实交易存在序列相关、setup 异质性和市场 regime 变化，因此 380 笔仍只是粗略下限。

**证据等级**：S 支持统计显著性需要样本量；U 支持当前具体数值表述。  
**来源状态**：目标文档直接检查 + 统计假设推演。

## 1.5 证据等级定义需要处理 preprint / working paper / 监管实验

**位置**：`foundation_2026.md:7`, `foundation_2026.md:89-97`, `foundation_2026.md:180-184`  

文档把 Bastani PNAS、监管实验、working paper、arXiv、Stanford SCALE、MIT EEG 等并列使用。方向可以，但证据等级应更细：同行评审、监管 RCT、工作论文、实验室研究、商业白皮书不能都算 S。

建议在文档开头新增：

> S-peer = 同行评审；S-reg = 监管 / 官方 RCT；M-wp = working paper / preprint；W = 商业或实践观察；U = 推演。

**证据等级**：来源分级治理建议为 U。  
**来源状态**：基于项目 `meta-rule.md` 与目标文档证据标注一致性检查。

# 视角 2：5+ 年长期盈利实战交易者

## 2.1 “开仓必须同时挂止损单”在 A 股 / 跳空 / 涨跌停 / 流动性差场景不总是可执行

**位置**：`foundation_2026.md:370-376`  

“开仓前定义止损”是对的；“必须同时挂止损单才算合法记录”可能过具体。A 股 T+1、涨跌停、隔夜跳空、期货滑点、集合竞价、部分券商条件单可靠性，都可能让“同时挂单”与“实际可控风险”不是一回事。

建议把 Plan 级约束改成：

> 每笔交易必须有 binding pre-commitment 的退出机制；具体形态可以是真实止损单、条件单、时间止损、仓位减半规则或不可撤销的风险预算锁定，由 Phase 2 按市场选择。

**证据等级**：M/S 支持预承诺和止损纪律；W 支持具体交易场景限制。  
**诚实标记：** 场景限制来自交易实践常识，未经本仓库内数据验证。

## 2.2 产品过滤器不能简单等于“衍生品禁区”，因为用户已经交易期货

**位置**：`foundation_2026.md:72-73`, `foundation_2026.md:270`, `foundation_2026.md:373`  

文档把 0DTE / CFD / F&O / 高杠杆永续列为禁区是合理的保护方向。但项目用户画像包含期货，仓库已有一笔 MGC 期货交易记录。如果“期货”被整体归为禁区，会与用户真实行为冲突。

建议改为分级产品过滤器：

1. **红区**：0DTE、超高杠杆永续、无止损机制产品、无法理解保证金机制的产品。
2. **黄区**：低杠杆、明确合约规格、可计算最大损失的小型期货合约；必须满足额外门禁。
3. **绿区**：低成本被动 ETF、低杠杆现金股票。

**证据等级**：M/S 支持高杠杆散户风险；U 支持三色分级方案。  
**来源状态**：基于目标文档与仓库 `trades/2026/05/2026-05-04-mgc-202606-long.md` 的一致性检查。

## 2.3 单一 setup 聚焦方向正确，但需要定义“第一个 setup 的禁止项”

**位置**：`foundation_2026.md:152-158`, `foundation_2026.md:274`  

“单一 setup 聚焦”是实战上少数真正可落地的训练方式。但文档只说应该集成，没有定义第一阶段 setup 的约束。对小白而言，第一个 setup 不应是高频、盘口 scalping、新闻追单、财报赌方向或高杠杆。

建议 Phase 2 明确：

> 第一个 setup 必须低频、可复盘、可定义失效条件、可控制滑点，并且每周交易次数有上限。

**证据等级**：W/U。  
**诚实标记：** 这是基于实战风控与训练可控性的推演，不是直接学术结论。

## 2.4 缺少交易成本 / 滑点 / 断连 / 平台故障作为 Plan 级约束

**位置**：`foundation_2026.md:492-525`  

文档对心理、AI、HCI 证据很重，但实盘中“系统能不能活下来”还取决于交易成本、滑点、手续费、汇率、合约换月、夜盘、保证金调整、平台断连。它们不只是 Design 细节，而是 Plan 级必须承认的约束。

建议新增：

> 所有训练结果必须记录并区分毛盈亏、净盈亏、滑点、费用、融资/隔夜成本；任何 edge 判断必须用净值而非毛值。

**证据等级**：M/W。  
**诚实标记：** 交易成本影响收益是金融市场基本常识；具体字段方案需 Phase 2 设计。

## 2.5 退出协议要区分“策略无效”与“执行无效”

**位置**：`foundation_2026.md:400-409`  

“连续 6 月跑输被动基准 + 心理痛苦 ≥ 7/10”作为退出条件有价值。但实战上需要区分：

- 策略本身没有 edge；
- 有 edge 但用户执行不了；
- 样本不足；
- 市场 regime 改变；
- 训练资金太小导致成本吞噬 edge。

建议退出协议输出不只一个“切换被动”，而是分叉：

1. 暂停交易；
2. 降低频率 / 降低杠杆；
3. 换 setup；
4. 完全转被动。

**证据等级**：U。  
**诚实标记：** 这是基于实盘诊断逻辑的推演。

# 视角 3：HCI / mHealth / 行为改变工具研究者

## 3.1 “用户满意度与学习效果负相关”表述过硬

**位置**：`foundation_2026.md:181`, `foundation_2026.md:462-485`  

文档把“用户满意度与学习效果负相关”写成较强结论。更稳妥的表达是：在某些学习反馈场景中，降低满意度的困难反馈可能提升学习效果；但这不等于所有工具满意度都与效果负相关。

建议改为：

> 在需要纠错和深度反思的学习任务中，用户即时满意度可能与长期学习效果背离；因此不应把满意度或 DAU 作为唯一优化目标。

**证据等级**：S/M 支持“检索练习、困难反馈、满意度不等于学习”；U 支持广义“负相关”。  
**诚实标记：** 这是对跨研究外推强度的收缩建议。

## 3.2 字段数在文档中仍有 6 / 10 / Phase 2 三种口径

**位置**：`foundation_2026.md:200`, `foundation_2026.md:217`, `foundation_2026.md:260`, `foundation_2026.md:453`, `foundation_2026.md:469`  

文档一处说字段砍到 ≤6，一处说字段 ≤10，一处又说具体数字 Phase 2 决定。这会让 Phase 2 无法判断“≤6”是证据约束还是设计候选。

建议统一为：

> Plan 级约束：必须设置字段上限，并通过低摩擦完成率与信息充分性测试确定。≤6 / ≤8 / ≤10 都是 Phase 2 候选。

**证据等级**：来源为目标文档内部一致性检查。  

## 3.3 Honesty oath 需要防止“承诺疲劳”

**位置**：`foundation_2026.md:197`, `foundation_2026.md:312`, `foundation_2026.md:376`, `foundation_2026.md:436`  

事前诚实承诺是低成本、高潜力设计。但每笔记录都显示同一句 oath，长期可能被用户机械跳过，变成“我知道了”的点击噪音。

建议 Phase 2 把 oath 交互形态列为重点测试：

- 勾选框；
- 手打一短句；
- 随机抽样显示；
- 周度承诺 + 每笔轻量确认；
- 违规后加重承诺。

**证据等级**：S 支持事前承诺一般有效；U 支持承诺疲劳风险。  
**诚实标记：** “承诺疲劳”是基于 HCI 摩擦与习惯化的推演，需实测。

## 3.4 反馈时机应成为 Plan 级约束

**位置**：`foundation_2026.md:347-362`, `foundation_2026.md:420-439`  

文档规定反馈组件不能做裁判、不能拟人化、不能谄媚，但没有充分区分“交易执行瞬间”和“复盘阶段”。交易瞬间只应做极简阻断；复盘阶段才适合解释。

建议新增约束：

> 执行期反馈必须短、硬、低解释；反思期反馈可以长、结构化、有证据链。

**证据等级**：M/U。  
**诚实标记：** 这是基于认知负荷与交易场景上下文切换的逻辑推演。

## 3.5 入口形态调研需要加入“交易工作台上下文切换成本”

**位置**：`foundation_2026.md:530-538`  

文档已经要求 Phase 2 做入口形态调研，这是优点。但还应明确评估：交易者真实工作台通常在券商终端、TradingView、手机 App、微信群/新闻源之间。要求跳到 Cursor / CLI / 文档系统，可能直接破坏 critical threshold。

建议新增调研问题：

> 决策链入口是否能出现在用户实际下单前的物理路径上？如果不能，系统会不会只记录“愿意记录的交易”，而漏掉最危险的冲动交易？

**证据等级**：U。  
**诚实标记：** 这是基于工作流摩擦的推演。

# 视角 4：临床心理学家（CBT / ACT 背景）

## 4.1 高反刍用户不能只靠“周报代替逐笔深挖”

**位置**：`foundation_2026.md:32-35`, `foundation_2026.md:526-528`  

文档已经识别 Sbarra、Watkins & Teasdale 的反向证据，这是重要优点。但对高反刍用户的处理仍停留在“是否有特殊 workflow”的待定项。鉴于交易涉及金钱亏损、自责和即时反馈，这应提升到 Plan 级安全约束。

建议新增：

> 系统必须有反刍 / 焦虑 / 赌博倾向筛查；高风险用户默认进入低频复盘、非逐笔深挖、禁止夜间复盘的安全模式。

**证据等级**：S 支持反刍风险；U 支持具体安全模式。  
**诚实标记：** 安全模式设计需临床专家二次 review。

## 4.2 “诚实”措辞需要避免滑向自我惩罚

**位置**：`foundation_2026.md:321-327`, 也影响原系统 `trade-journal-template.md:8` 的语气  

项目长期强调“诚实”，这是核心资产。但“不要骗自己”“对自己撒谎比亏钱更致命”对完美主义、强迫倾向、抑郁倾向用户可能变成自我攻击。

建议把主语从“你是不是诚实的人”改成“这条记录是否足够可检验”：

> 我们审查的是记录质量，不审判人格。

**证据等级**：M/S 支持 self-compassion / CBT 中避免过度自责的重要性；U 支持具体文案修改。  
**诚实标记：** 我没有在本次 review 中核验 Neff / Gilbert 原文，建议 Phase 2 临床安全 review 补证据。

## 4.3 Problem gambling 倾向应成为明确禁忌症

**位置**：`foundation_2026.md:542-552` 局限部分未覆盖；`foundation_2026.md:400-409` 退出协议也未显式覆盖  

主动交易训练与问题赌博在“下注、不确定奖励、亏损后追赶、即时反馈”上有重叠。系统若只把用户定义为“想成为交易者”，可能漏掉“不适合被鼓励继续训练”的人群。

建议新增绝对禁忌或暂停条件：

- 借钱交易；
- 隐瞒家人交易；
- 亏损后加码翻本；
- 因交易影响睡眠 / 工作 / 家庭；
- 无法停止看盘；
- 出现自伤意念或严重绝望。

**证据等级**：M/S（DSM-5 problem gambling 框架方向）；U（具体触发列表需临床核验）。  
**诚实标记：** 需要临床专业人士确认触发条件措辞。

## 4.4 “Future self”工具不应默认安全

**位置**：`foundation_2026.md:201`, `foundation_2026.md:314`  

未来自我工具可能帮助长期主义，但对抑郁、羞耻感强或财务压力高的人，可能触发“我未来也没希望”的反向反应。

建议改为：

> Future self 是候选工具，不是默认 S 级硬组件；上线前必须定义适用人群、禁用条件和退出提示。

**证据等级**：S/M 支持 future self 相关研究存在；U 支持交易训练安全性迁移。  
**诚实标记：** 这是临床风险迁移推演。
